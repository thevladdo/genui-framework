# GenUI Output Guarantees Statement

**Audience**: the legal, compliance or security team of an entity running a GenUI deployment. This document is written to be attached to a contract.

**Scope**: every piece of LLM-generated UI served by one GenUI deployment: zone renders (`/zone/render`, `/zone/render/stream`, `/zone/batch-render`, warmup, background refreshes) and chat responses (`/query`).

**Claim**: the properties below are enforced by code that runs **after** generation, on **every** serving path. They hold regardless of what the model outputs, which model is plugged in, or how it is prompted.

**As of**: 2026-07-20. Line references are to this revision; each row names the symbol so it can be re-verified with `grep` after updates.

## The one principle everything derives from

**A guarantee is a check on the output, never an instruction to the model.**
Prompt instructions shift probability; they bound nothing, and their violations are invisible. Every guarantee in the table below is a post-generation gate with a test that goes red if the gate is removed: the recorded model responses in `backend/tests/golden/` are deliberately adversarial (invented links, invented prices, missing pinned items), so the suite fails if any link of the chain stops working.

The chain, in order, on all three paths (zone sync, zone SSE, chat):

```
schema validation → URL whitelist → numeric grounding → content policy → pinned enforcement
```

## Honest limits: what is best-effort, not guaranteed

A guarantee that overpromises is worse than none. The following are real limits of the mechanisms above:

- **Numeric grounding is verbatim, not semantic.** It guarantees the digits shown existed in the input; it does not verify the _claim around_ the number. "120 countries" with 120 in the input passes even if the input said "120 employees". Magnitude conversion is not attempted: a model that rewrites "10,000,000" as "10M" loses that stat (removed, reported). The model is instructed to copy figures as-is; the instruction is the quality lever, the guard is the guarantee.
- **Numbers in prose are not checked.** Grounding applies to fields whose value IS the number (stat values, prices, chart points). Digits inside descriptions, labels or markdown text are not enforced: stripping them would mangle legitimate prose. If a number must be guaranteed, put it in a `stats_banner` / `pricing_cards` / `chart`, or pin it.
- **Tone is not enforceable.** `CONTENT_POLICY` enforces term absence, which a regex can prove. Tone constraints ("formal", "no superlatives") remain prompt-level instructions: real influence, no guarantee. Nothing in this document claims otherwise.
- **Banned-term matching is lexical.** Word-boundary and case-insensitive, but not semantic: misspellings, synonyms or paraphrases of a banned term are not caught. The blocklist guarantee is "this exact term never appears", not "this topic never appears".
- **Text content of allowed components is model-authored.** Outside the enforced fields above, wording is the model's (that is the product). The measurement loop (`/events/stats`) and the audit trail exist to observe it, and pinned content exists to fix what must be exact.

## How to re-verify this document

```bash
cd backend && python3 -m unittest discover -s tests           # full suite
python3 -m unittest tests.test_output_guarantees -v            # this document's chain
python3 -m unittest tests.test_golden_zone -v                  # adversarial end-to-end
```

The golden harness replays deliberately hostile model output through the real pipeline. Neutering any guard in the chain turns it red.
